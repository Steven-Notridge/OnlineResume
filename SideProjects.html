PowerShell

Let me just start, by admitting that PowerShell is technically my first love, at least for scripting... It all started with a simple mistake that Engineers were making, when creating new user accounts. I just sat there for a moment and thought to myself, you know... This is technically the 'future' right? Surely this can be automated! I researched it, tested it, and before I knew it, the client approved it and it became the main method of user creations and no further issues happened again!

Python

Hmm. Where to start? The easiest language to learn? Are the rumours true? I'd say so. I've found some languages like JavaScript a bit annoying to figure out, but Python just sits well. Whilst I'm not as proficient with it as PowerShell, I'm definitely comfortable reading it all. The main question I found myself asking however, how do I learn this though? Sure, people use Python for all sorts of things, and I even saw someone drawing art with it. Even though I tried to replicate that, I failed. I found that this was mostly because of the way libraries are used and referenced. It's kind of strange using an outdated or redundant library and trying to use it in code. PowerShell doesn't really have modules that expire, or stop working because of a new version. So this is something I need to wrap my head around, albeit it's not easy when nobody you know enjoys this stuff!

Linux

Debian or else. That is all. Jokes aside, it's my daily driver. Once I got down all of the strange issues at the start, I grew to love this distro. I originally tried Manjaro but the Networking just didn't work! So I gave up and tried Debian, and it just worked... Kinda. After I uninstalled xserver-xorg-video-intel and installed firmware-misc-nonfree, it was like the icing on the cake. Perfection. Buttery smooth animations at last. So I forced myself to use this as much as possible, how else would you learn? What did I do next, you ask? Well, I for some reason chose violence. Pulled out an old desktop, installed Debian ServerOS with no GUI, and decided, "Hey, why not see what KVM is like?" Ahh... It took a few days but I managed to recreate my workplaces Windows infrastructure with it. It didn't run very well, god no. But it worked. Every tutorial or help article out there was insisting on using the GUI, but I didn't want that. It took more time finding the right commands than actually doing anything else. I already know Windows well enough. But the best part? I ran this command, route add default gw 192.168.0.78 and yep, it nuked my external access and left me dumbfounded for hours. You know something's gone terribly wrong when you can't resolve ftp.uk.debian.org!

I'm going to add a couple details about KVM, in case someone needs it. But the below is the config I used for my Windows Servers, and this is what I saved for the initial one. You need to save this as a .sh and then run it from your KVM server. It may also be less pain if you make another script for startup that runs virsh net-start default to prevent error: Cannot get interface MTU on 'virbr0': No such device.

virt-install \ --name Server2019 \ --description "Windows Server 2019 Template" \ --os-type=windows \ --memory 8192 \ --vcpus=4 \ --cdrom /home/Steven/Iso/WindowsServer2022.iso \ --disk path=/var/lib/libvirt/images/windows2019.qcow2,bus=virtio,size=30 \ --disk /home/Steven/Iso/virtio-win.iso,device=cdrom,bus=ide \ --graphics vnc,listen=0.0.0.0 \ --noautoconsole \ --network bridge=virbr0,model=virtio

You will need to download the virtio iso, and then load the drivers under the amd64 folder during the installation. After you've booted to Windows, you then need to go into Device Manager and then configure it further.
